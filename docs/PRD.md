# 產品需求文件 (PRD) - 線上算命系統

## 1. 專案概述
### 背景與動機
在現代社會中，許多人面臨生活、工作、感情上的壓力與迷惘，對於未來的發展感到不確定。傳統的算命方式需要親自前往相命館，較不方便且有時間限制。本專案期望透過「線上算命系統」，讓使用者隨時隨地都能獲得初步的命理建議與心靈撫慰，將傳統文化與現代科技結合。

### 目標用戶
任何對命理、占卜有興趣，或是近期感到迷惘、需要指引的大眾。

### 核心價值主張
提供快速、方便且具有神秘感的線上算命與占卜服務，幫助使用者了解自己的生辰八字，並針對特定問題提供占卜結果與指引。

## 2. 功能需求
以下為系統主要功能及其使用者故事：

### (1) 生成八字
- 作為 **一般使用者**，我希望 **輸入我的出生年、月、日、時**，以便 **系統能自動換算並顯示我的生辰八字（天干地支）**。

### (2) 算命 (命格解析)
- 作為 **一般使用者**，我希望 **系統能根據我的八字提供整體的命理分析**，以便 **我了解自己的性格優缺點與未來運勢**。

### (3) 占卜 (解答解惑)
- 作為 **一般使用者**，我希望 **能在心中默想具體問題後進行線上抽籤或占卜**，以便 **針對特定疑惑（如感情、事業、財運）獲得解答與指引**。

### (4) 歷史紀錄檢視
- 作為 **一般使用者**，我希望 **系統能保存我過去的算命與占卜結果**，以便 **我日後隨時回顧與比對**。

### (5) 結果分享
- 作為 **一般使用者**，我希望 **能將算命或占卜結果匯出或產生連結**，以便 **分享給親友或社群媒體**。

## 3. 非功能需求
### 技術限制
- Web 框架：Flask (Python)
- 樣板引擎：Jinja2
- 前端技術：HTML, CSS, JavaScript
- 資料庫：SQLite

### 效能與安全考量
- **效能**：八字排盤與資料查詢的計算速度需即時，確保頁面載入時間小於 2 秒。
- **安全**：
  - 驗證使用者輸入的時間格式，避免無效資料與潛在的注入攻擊。
  - 對於有紀錄儲存需求的功能，需確保每個 Session/帳號間的資料隱私隔離。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須要有 - 核心功能)**：
  - 生成八字功能
  - 基本的算命結果展示
  - 基礎線上占卜（例如抽籤）
- **Should Have (應該要有 - 重要功能)**：
  - 歷史紀錄檢視（基於 Session 或簡易會員系統）
- **Nice to Have (有會更好 - 加分項目)**：
  - 精美的命盤視覺化顯示
  - 占卜結果一鍵分享至社群平台
